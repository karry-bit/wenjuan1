# 问卷一信息收集与关键词分析系统功能说明

## 一、系统定位

本系统用于承接“银发群体防骗经验认知与策略研究”中的问卷一，重点解决两个问题：

1. 把问卷一的基础信息和开放式回答稳定收集下来。
2. 把开放题中的经验内容做初步统计和主题提炼，避免只收集不分析。

问卷一本质上不是普通的封闭式量表，而是偏质性、偏经验提取的开放式问卷。因此系统设计上采用“问卷录入 + 关键词分析 + 主题归类 + 原始记录回看”的组合方式。

## 二、当前已实现功能

### 1. 问卷填写与提交

- 支持录入基础信息：
  - 性别
  - 年龄段
  - 居住情况
  - 主要信息来源
- 支持录入问卷一全部开放题内容。
- 基础信息未填完整时，系统会阻止提交。
- 为保证后续分析有效，系统要求开放题至少填写 3 题以上。

### 2. 本地数据存储

- 每次提交后，数据会保存到浏览器本地 `localStorage`。
- 关闭页面后，当前浏览器里的数据不会立即丢失。
- 适合演示、预调研、小规模录入和前期试用。

### 3. 样本统计

- 自动统计样本总数。
- 自动统计平均文本长度。
- 自动统计：
  - 性别分布
  - 年龄分布
  - 居住情况分布
  - 信息来源分布

### 4. 热词分析

- 系统根据问卷一主题预置了中文关键词词典。
- 当前会自动识别并统计五类关键词：
  - 风险信号
  - 核实动作
  - 谨慎原则
  - 经验来源
  - 支持需求
- 页面会展示：
  - 热词云墙
  - 关键词排行

### 5. 主题归类分析

- 系统会统计每个主题被多少份问卷提及。
- 系统会统计主题的关键词总命中次数。
- 系统会给出每个主题的代表词，方便后续人工编码或写研究结论。

### 6. 自动洞察

系统会根据当前数据给出简要洞察，例如：

- 当前样本最集中的年龄段
- 当前最常见的信息来源
- 哪类主题最容易被受访者提及
- 哪个关键词最值得继续追问
- 是否已经出现“子女提醒”“紧迫话术”等研究重点

### 7. 题目完成度检查

- 系统会按题目统计完成率。
- 系统会按题目统计平均文本长度。
- 可以快速发现哪些题最容易引出有效叙述，哪些题后续需要优化提问方式。

### 8. 原始记录回看

- 页面会展示最近提交的问卷摘要。
- 点击后可查看该份问卷的主要原始答案内容。
- 便于研究团队快速抽样阅读和人工复核。

### 9. 数据导出

- 支持导出 `JSON`
- 支持导出 `CSV`

其中：

- `JSON` 适合后续继续开发系统或导入程序处理。
- `CSV` 适合导入 Excel、SPSS 或手动整理。

### 10. 示例数据演示

- 支持一键加载示例数据。
- 用于展示词云、排行、主题归类的页面效果。
- 适合老师汇报、方案展示和前期试运行。

## 三、系统页面结构

### 1. 顶部说明区

- 展示系统定位
- 展示我对问卷一的主动分析
- 让使用者一进页面就知道这不是普通问卷录入页，而是“采集 + 分析”页面

### 2. 左侧问卷填写区

- 基础信息
- 开放题填写
- 提交与清空

### 3. 右侧分析区

- 本地数据操作
- 样本核心指标
- 结构分布图
- 热词云墙与关键词排行
- 主题归类分析
- 自动洞察
- 题目完成度统计
- 最近提交内容

## 四、分析逻辑说明

### 1. 为什么要做关键词分析

因为问卷一主要是开放题。如果只收集文本，不做归类，后续整理成本会很高，研究价值难以快速显现。

### 2. 为什么不用复杂模型，先用主题词典

当前版本是第一版可用系统，目标是先快速落地。对于问卷一这种前期探索工具，先用主题词典做热词统计和主题扫描，已经能支持：

- 预调研
- 结构化观察
- 后续访谈提纲修订
- 后续量化问卷题项提炼

### 3. 当前主题词典围绕什么建立

主题词典围绕问卷一核心研究目的建立，重点是提取：

- 老人如何发现异常
- 老人如何核实真假
- 老人有哪些防骗底线
- 老人的经验来源来自哪里
- 老人希望得到什么支持

## 五、使用方式

1. 打开 `index.html`
2. 直接填写并提交问卷
3. 在右侧查看统计、热词和主题分析
4. 需要演示时可以加载示例数据
5. 需要进一步处理时导出 JSON 或 CSV

## 六、当前版本边界

- 当前版本是前端单机版，不带后端数据库。
- 数据默认保存在当前浏览器，不会自动同步到别的设备。
- 关键词分析采用预置词典，适合前期探索，不等于最终正式编码结果。

如果后续要扩大到大规模调研，建议增加服务端存储、账号权限、批量导入和更细的文本编码能力。
